Section I: Bezier Curves and Surfaces

Part 1: Bezier Curves with 1D de Casteljau Subdivision

Briefly explain de Casteljau's algorithm and how you implemented it in order to evaluate Bezier curves.

The de Casteljau algorithm finds the points at the ratio t between all points, and it does this repeatedly until there is only one point left. This point will be on the resulting Bezier curve at parameter t. To implement one step of this algorithm, we iterate through all the points and use linear interpolation to find new points. For given point pi, pi+1, and t, the new point p would be at pi + t * (pi+1 - pi).

Part 2: Bezier Surfaces with Separable 1D de Casteljau

Briefly explain how de Casteljau algorithm extends to Bezier surfaces and how you implemented it in order to evaluate Bezier surfaces.

We use the separable 1D de Casteljau, as defined in lecture. We can first apply the de Casteljau algorithm in dimension u to define control points for the moving curve in dimension v, and then we can apply the de Casteljau algorithm in v to get the final point.

We implemented this by using three separate functions. evaluateStep() performs one step of de Casteljau, like in task 1, and evaluate1D() runs de Casteljau fully in one dimension by calling evaluateStep until there is only 1 point left. evaluate() runs de Casteljau fully in 2 dimensions to evaluate the Bezier patch at parameter (u, v). It calls evaluate1D() for all of u (all the rows) to get the moving curve, and then, it finds the final point by calling evaluate1D() for v on the points of the moving curve.

Section II: Triangle Meshes and Half-Edge Data Structure

Part 3: Area-Weighted Vertex Normals

Briefly explain how you implemented the area-weighted vertex normals.

To calculate the area-weighted vertex normals, we used the formula from lecture where the vertexNormalVector = (the sum of the facesNormalVectors) / (the euclidean norm of the sum of the facesNormalVectors). We also had to weight the normals of the faces by their area, and we found this area by using the cross product of two of the face’s vectors to get a normal vector. The magnitude of this normal vector is proportional to the area of the face. We were already provided a method to find the normal vector of a face, so we just multiplied the magnitude/euclidean length of that vector with itself. We then did this for every face that touches the vertex and added them together. Then, we just divided by the norm to get a normalized unit vector.

Part 4: Edge Flip

Briefly explain how you implemented the edge flip operation and describe any interesting implementation / debugging tricks you have used.

We followed the strategy of the image below in order to accomplish the flip operation. We first set pointers to all of the items in the two triangles: 6 inside halfedges, 4 outside halfedges, 2 faces, 4 vertices, 5 edges. We kept the locations of the inside halfedges in relation to each other the same. It’s like we took all the inside halfedges and rotated them clockwise. The faces stayed the same too because we made the left face the new top face. What we changed were the locations of the halfedges, and we did this by changing the vertices, edges, and twins they were associated with. For example, h2’s vertex would change from v2 to v0, its edge from e2 to e3, and its twin from h7 to h8, and its next and face would stay the same. We went through all the halfedges and changed their next, twin, vertex, edge, and face pointers like the below image using setNeighbors. Then, we went through all the vertices, edges, and faces and changed their halfedge pointer to be the new one they are associated with.
